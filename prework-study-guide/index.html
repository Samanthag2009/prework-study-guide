HTML

  • HTML is the "framework" of a website
  • The "Head" contains metadata that helps render the page for the browser but isn't usually seen by the user
  • The "Body" is full of elements that the user can see
  • An "Element" is a bracketed piece of information that serves a purpose to the webpage code
  • An "Attribute" defines additional information about the element. They are defined in the beginning element, but never the closing element.
  • When multiple elements are arranged within one element, it is called "nesting"
  • Elements that are nested within another element are called "child elements"
  • The element that contains the child elements are called "Parent Elements"
  • An element refers to all the infomation between opening and closing, but the bracketed pieces of the element are called "tags"
  • Closing tags will contain a forward slash

Git

  • Git Commands are always preceded by "git" in the command line
  • Git is a version control software that allows us to create repositories, initiate repositories, add code to the repositories, and view versions of the code with their own commits
  • The Git states are "Modified," "Staged," and "Commited"
  • "Modified" is a changed file but not commited to the database
  • "Staged" is a file marked in the current version to be included into the commit "snapshot"
  • "Commited" is when a file is safely stored in the local database
  • git status: command to show modified files in the working directory staged for commit
  • git add [file]: add a files as it looks now to the next commit (stage)
  • git commit -m "[descriptive message]": Commit your staged content as a new commit snapshot
  • git checkout: switch to another branch and check it out into the working directory
  • git branch [branch-name]: create a new branch at the current commit
  • git branch: list your branches * will appear next to the currently active branch
  • git checkout -b [branch-name] creates a new branch with the given name and switches to it
